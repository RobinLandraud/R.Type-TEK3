\doxysubsection{Detailed Description}
Chai\+Script representation of std\+::string. It is an std\+::string but only some member are exposed to Chai\+Script. 

Because the Chai\+Script string object is an std\+::string, it is directly convertible to and from std\+::string using the \mbox{\hyperlink{namespacechaiscript_a2de45834af1787f63a901a871414b749}{chaiscript\+::boxed\+\_\+cast}} and \mbox{\hyperlink{namespacechaiscript_a7f9fcf9713fa8a33bcc304a16ac2834b}{chaiscript\+::var}} functions.

With the exception of \mbox{\hyperlink{classChaiScript__Language_1_1string_acd8c33a93989c9c4c89c8b8112c041ea}{string\+::trim}}, \mbox{\hyperlink{classChaiScript__Language_1_1string_ab0133519272536216eac6fa3c8845df7}{string\+::rtrim}}, string\+::ltrim, all members are direct pass-\/throughs to the std\+::string of the same name.

\begin{DoxyNote}{Note}
\mbox{\hyperlink{classChaiScript__Language_1_1Object}{Object}} and function notations are equivalent in Chai\+Script. This means that {\ttfamily \char`\"{}bob\char`\"{}}.find(\char`\"{}b\char`\"{}) and {\ttfamily find}(\char`\"{}bob\char`\"{}, \char`\"{}b\char`\"{}) are exactly the same. Most examples below follow the second formation of the function calls. 
\end{DoxyNote}
